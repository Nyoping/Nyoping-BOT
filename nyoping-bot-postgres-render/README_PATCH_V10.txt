뇨핑봇 패치 V10

핵심 수정
1) 역할 검색 API가 role_id를 숫자가 아니라 문자열로 반환
- 브라우저 JS의 안전 정수 범위(2^53-1) 문제로 Discord snowflake ID가 반올림되어 저장되던 문제 수정
- 이 문제는 반응 역할 / 반응 차단 / 레벨 역할에 공통으로 영향 가능

2) 레벨 역할 선택 UI에서 Number(roleId) 제거
- 레벨 역할 저장 시 role_id를 문자열로 유지

3) 레벨 역할 저장/삭제 후 전체 멤버 역할 동기화 큐 자동 등록
- 대시보드에서 규칙 바꾼 뒤 봇이 순차적으로 실제 서버 역할을 다시 맞춤

중요
- 이전 버전에서 저장된 규칙은 role_id가 이미 깨졌을 수 있습니다.
- 이번 패치 적용 후에는 아래를 꼭 해주세요.
  1. 레벨 역할 규칙 전부 삭제 후 다시 저장
  2. 반응 역할 규칙 전부 삭제 후 다시 저장
  3. 반응 차단 규칙도 삭제 후 다시 저장 권장

적용
- Render: nyoping-bot-postgres-render 전체 덮어쓰기 후 재배포
- weirdhost: 같은 ZIP 기준으로 전체 덮어쓰기 후 재시작 권장
