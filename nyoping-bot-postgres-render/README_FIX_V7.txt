뇨핑봇 Render+weirdhost 통합 핫픽스 v7

이번 버전에서 고친 핵심
1) 역할 검색 API가 Discord 역할 ID를 JSON 숫자로 내려주던 문제 수정
   - Discord ID(18~19자리)는 브라우저 JS Number로 깨질 수 있음
   - 이제 역할/유저 ID를 문자열로 내려주도록 고침
   - 레벨 역할 규칙 / 반응 차단 / 반응 역할 저장 시 잘못된 ID가 들어가던 문제 완화

2) 레벨 역할 규칙 저장 후 실제 서버 반영이 늦거나 안 되던 문제 보강
   - Render 대시보드에서 규칙 저장/삭제 시 role_sync_queue에 현재 서버 멤버들을 자동으로 넣음
   - weirdhost 봇의 role sync worker가 순차 반영함

3) 반응 역할 현재 규칙에서 이모지가 안 보이던 문제 수정
   - admin.html에서 rr.emoji -> rr.emoji_key로 수정

4) 반응 역할 삭제 422 오류 수정
   - 삭제 폼이 emoji_key를 보내도록 수정
   - 백엔드도 emoji_key / emoji 둘 다 받도록 보강

5) 반응 차단 / 반응 역할이 바로 테스트하면 안 먹는 문제 보강
   - weirdhost 봇이 raw reaction 이벤트를 받았을 때 캐시에 규칙이 없으면 DB에서 해당 길드 규칙을 즉시 다시 불러오도록 수정
   - 주기적 갱신도 더 짧게 조정

6) 봇이 관리할 수 없는 역할 하나 때문에 전체 역할 수정이 실패하던 문제 완화
   - 봇보다 높은 역할/@everyone 역할은 건드리지 않고 가능한 역할만 적용하도록 보강

매우 중요
- 이전 버전에서 이미 "깨진 역할 ID"로 저장된 레벨 역할 규칙은 DB에 남아 있을 수 있습니다.
- 적용 후에는 대시보드에서 기존 레벨 역할 규칙/반응 역할 규칙을 한번 삭제하고 다시 저장하는 것을 권장합니다.
- 특히 /레벨역할 목록에서 "알 수 없는 역할"이 보였던 기존 규칙은 다시 저장해 주세요.

적용 권장 순서
A. Render
1. 이 ZIP의 nyoping-bot-postgres-render/ 전체를 Render 소스 기준으로 교체
2. 재배포
3. /admin 접속
4. 기존에 잘못 저장된 레벨 역할 규칙 / 반응 역할 규칙이 있으면 삭제 후 다시 생성

B. weirdhost
1. 이 ZIP의 nyoping-bot-postgres-render/ 기준으로 업로드
2. 최소한 nyopingbot/ 폴더 전체 + bot_entry.py 를 교체
3. 서버 재시작
4. 봇 로그인 확인

적용 후 테스트 순서
1. 레벨 역할 규칙 새로 저장
2. /레벨역할 목록 확인
3. 반응 차단 새로 저장 후 실제 반응 눌러보기
4. 반응 역할 추가/제거 새로 저장 후 실제 반응 눌러보기

주의
- 반응 차단은 "누르자마자 클릭 자체를 막는 기능"이 아니라 "누르면 곧 제거" 방식입니다.
- 반응 차단에는 봇 권한 Manage Messages가 필요합니다.
- 역할 추가/제거/레벨 역할에는 봇의 Manage Roles 권한과 역할 위치가 대상 역할보다 높아야 합니다.
